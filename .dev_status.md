### Phase 30: AI Pool Routing Fix
**Lỗi**: AI vẫn call model mặc định thay vì model trong pool (`agent_routing.json`).

**Root cause**: `RotateKeyAction` lọc `where('metadata->model', $model)` quá chặt — key trong `ai_key_pool` có `metadata` rỗng nên không bao giờ match → pool exhausted 100% → fallback lên config file/local với model mặc định.

**Fix**: Chỉ lọc model khi key có model trong metadata. Key không có model thì match với mọi request:
```php
$query->where(function ($q) use ($model) {
    $q->whereRaw("metadata->>'model' = ?", [$model])
      ->orWhereRaw("metadata->>'model' IS NULL")
      ->orWhereRaw("metadata = '[]'");
});
```

**Copy + restart**: `RotateKeyAction.php` vào 3 containers PHP + restart.
